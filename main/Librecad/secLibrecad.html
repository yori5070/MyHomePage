LibreCAD ３ をコンパイルしてみました　--- 2017年６月 ---

　librecad 3 を試行錯誤のすえにコンパイルすることができました。LibreCAD ３ は BRL-CAD （3次元CAD）のユーザーインターフェイスとして開発されています。まだまだ完成には程遠いようです。

　 https://github.com/LibreCAD/LibreCAD_3 に書かれているとおりに LinuxMint でコンパイルするための環境をつくります。

　まずライブラリを導入します（apt-get install qttools5-dev qttools5-dev-tools libqt5opengl5-dev liblua5.2-dev git g++ gcc-4.8 libcairo2-dev liblog4cxx10-dev libpango-1.0-0 libpango1.0-dev libboost-all-dev libqt5svg5 libgtest-dev libeigen3-dev libcurl4-gnutls-dev）。

　さらに「You need to compile Google Test in /usr/src/gtest/ and move the libraries in /usr/lib/」と書かれているますので、/usr/src/gtest にある Google Test をコンパイル（cd /usr/src/gtest/src; sudo cmake ..; make）し、ビルドできたライブラリ *.a を /usr/lib/ にコピーします。

　環境ができたら、LibDxfRW のソースを github からソースを clone、コンパイルてインストール。

　そして本体の LibreCAD のソースを clone し,コンパイルします。

LibreCAD ３ を起動時のエラー　--- 2018年９月 ---

　起動時にエラーは、環境変数に「QT_STYLE_OVERRIDE=gtk2」をセットすることで解決。
